## Практические работы по курсу «Искусственный интеллект»

Добро пожаловать в репозиторий с практическими заданиями по курсу **«Искусственный интеллект»**! Здесь вы найдёте материалы в формате Jupyter Notebook (`.ipynb`) для отработки ключевых тем: от основ машинного обучения до нейронных сетей. 

### Как устроен процесс работы:
1. **Скачайте** файл с нужной практикой (`PractX/ML_pract_X.ipynb`)
2. **Выполните** задания в [Google Colab](https://colab.research.google.com/)
3. **Сохраните** результат и отправьте работу через **Pull Request** в этот репозиторий
4. **Получите обратную связь** от преподавателя прямо в интерфейсе GitHub


> 💡 *Работа с этим репозиторием поможет вам не только освоить ИИ, но и получить ценный опыт использования инструментов современной разработки: Git, GitHub, code review и совместной работы над кодом.*

## **ИНСТРУКЦИЯ ДЛЯ СТУДЕНТА**

### Предварительные требования
- Аккаунт на [GitHub](https://github.com/)
- Аккаунт на [Google Colab](https://colab.research.google.com/) (достаточно аккаунта Google)
- Установленный браузер (рекомендуется Chrome/Edge/YandexBrowser)

---

### Шаг 1: Подготовка репозитория (один раз)

1. **Сделайте форк репозитория**
   - Откройте [https://github.com/kubenet/ML_course](https://github.com/kubenet/ML_course)
   - Нажмите кнопку **Fork** в правом верхнем углу
   - Дождитесь создания копии репозитория в вашем аккаунте

2. **Клонируйте форк на локальную машину**
   ```bash
   git clone https://github.com/ВАШ_НИК/ML_course.git
   cd ML_course
   ```

---

### Шаг 2: Получение задания

1. Откройте **ваш форк** репозитория
2. Перейдите в папку с нужной практикой: `Pract1/`, `Pract2/` и т.д.
3. Откройте файл задания (например, `ML_pract_1.ipynb`)
4. Нажмите кнопку **Raw** → скопируйте ссылку на файл ИЛИ
   - Нажмите **Open in Colab** (если настроено) ИЛИ
   - Скачайте файл: кнопка **Download**

---

### Шаг 3: Работа в Google Colab

1. Откройте [Google Colab](https://colab.research.google.com/)
2. Загрузите файл:
   - Вкладка **File** → **Upload notebook** → выберите скачанный `.ipynb`
   - *ИЛИ*: `File` → `Open notebook` → вкладка `GitHub` → вставьте ссылку на файл
3. Выполните все задания:
   - Запускайте ячейки по очереди (`Shift + Enter`)
   - Заполняйте пропущенные участки кода
   - Сохраняйте изменения: `File` → `Save a copy in Drive` или `Ctrl+S`

> ⚠️ **Важно**: Убедитесь, что все ячейки выполнены, результаты видны, и нет ошибок выполнения.

---

### Шаг 4: Сохранение и подготовка к отправке

1. После завершения работы:
   - `File` → `Download` → `Download .ipynb`
2. Переименуйте файл по шаблону:
   ```
   pract_1_Фамилия_Имя.ipynb
   ```
3. Поместите файл в соответствующую папку вашего локального форка:
   ```
   ML_course/Pract1/pract_1_Фамилия_Имя.ipynb
   ```

---

### Шаг 5: Отправка работы через Pull Request

1. **Закоммитьте изменения** (если работаете локально):
   ```bash
   git add Pract1/pract_1_Фамилия_Имя.ipynb
   git commit -m "Submit pract_1: Фамилия Имя"
   git push origin main
   ```

2. **Создайте Pull Request**:
   - Откройте **оригинальный репозиторий** [kubenet/ML_course](https://github.com/kubenet/ML_course)
   - Нажмите **Pull requests** → **New pull request**
   - Выберите: `base: main` ← `compare: ВАШ_ФОРК:main`
   - Убедитесь, что виден ваш файл с решением
   - Заполните описание:
     ```
     Практическая работа №1
     Студент: Фамилия Имя
     Группа: [ваша группа]
     Комментарий: Все задания варианта №42 выполнены, код протестирован
     ```
   - Нажмите **Create pull request**

3. **Дождитесь проверки**:
   - Преподаватель оставит комментарии или подтвердит принятие
   - При необходимости внесите правки и обновите PR (`git push` автоматически обновит PR)

---

### Чек-лист перед отправкой работы на проверку

- [ ] Все ячейки кода выполнены без ошибок
- [ ] Результаты вывода (графики, таблицы, метрики) сохранены в ноутбуке
- [ ] Файл назван корректно и помещён в нужную папку
- [ ] В описании PR указана фамилия, имя и группа
- [ ] Не изменены исходные файлы заданий (создайте копию для решения)
